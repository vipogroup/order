# תאריך כיוון סכום ₪ הערה

אין רישומים — הוסיפו שורה ראשונה ביומן.

איך לקרוא את המספרים

  • ניסים — «שילם אליי» מסוכם מול מחיר הסחורה (עלות מפעל ₪ לפי «מה הגיע»). אם הסחורה > מה שנכנס — מופיע חסר לכיסוי (הוא צריך להשלים אליך). אם נכנס > סחורה — נוצרת קרן לזכותו; כל «החזרה ממני» מפחיתה ממנה, ו־נותר להחזיר = מה שעדיין חייבים להחזיר לו.
  • דורון — אותו עיקרון מול השקעה (מה שהזנת). רווח על ההשקעה = מחזור המכירות של דורון במערכת מינוס ההשקעה — זה אינדיקטור כלכלי לפעילות, לא ייעוץ משפטי לחלוקת רווחים.

שיתוף נתונים בין מכשירים

כדי שכל מי שיש לו את הקישור יראה את אותם נתונים, צריך מסד בענן (Firebase Firestore — חינמי לשימוש בסיסי). אי אפשר לעשות זאת רק מתוך הקישור בלי שלב הגדרה אחד.

  1. נכנסים ל־Firebase Console → יוצרים פרויקט.
  2. בוחרים Firestore Database → יוצרים מסד (Production או Test).
  3. גלגל השיניים → Project settings → בוחרים אפליקציית Web → מעתיקים את בלוק ה־firebaseConfig (אובייקט JSON).
  4. ב־Firestore → Rules — מדביקים (לצוות פנימי בלבד; לא לפרסום רחוב):
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /vipo_state/{doc} {
      allow read, write: if true;
    }
  }
}

הנתונים בענן נשמרים תחת מזהה מסמך קבוע. קונפיג Firebase (להלן) נדרש פעם אחת למערכת — אפשר להטמיע אותו בקובץ js/firebase-config.js בפריסה כדי שלא יצטרכו להדביק JSON בכל טלפון.
